What are DDL and DML?
DDL (Data Definition Language) is used to define and manage the structure of the database. It deals 
with creating, modifying, or deleting the “blueprint” of tables and other objects. The main DDL 
command is CREATE, along with ALTER and DROP.
DML (Data Manipulation Language) is used to work with the actual data stored inside the tables. It 
allows us to add, change, or remove records. The key DML commands are INSERT, UPDATE, DELETE, 
and SELECT.
Main difference:
1.	DDL changes the structure of the database (e.g., creating a new table), while DML works 
with the data inside those tables. 
2.	DDL commands are usually permanent and affect the whole table, whereas DML changes 
can often be undone with a transaction rollback.

How filtering with the WHERE clause works
The WHERE clause is used with SELECT, UPDATE, and DELETE to return or affect only the rows that 
meet certain conditions. It makes queries precise and powerful. Some useful operators I used 
include:
*	= ? equal to (e.g., WHERE class = 'Form 3')
*	> or >= ? greater than / greater than or equal to (e.g., WHERE marks >= 80)
*	BETWEEN ? values within a range (e.g., WHERE marks BETWEEN 60 AND 79)
*	IN ? matches any value in a list (e.g., WHERE class IN ('Form 3', 'Form 4'))
*	LIKE ? pattern matching (e.g., WHERE first_name LIKE 'A%' for names starting with A)
Combining these with AND and OR allows very specific filtering.
